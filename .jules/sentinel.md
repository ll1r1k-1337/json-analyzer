## 2024-05-24 - Allocation-based Denial of Service (DoS) in Binary Reader
**Vulnerability:** The binary token parser (`BinaryTokenReader` and `FileReader`) unconditionally allocated memory blocks based on length fields supplied within the input stream. An attacker could craft a malicious or corrupted binary token (e.g., a `Number` token with a massive length prefix) that would force `Buffer.allocUnsafe` or `Buffer.alloc` to request gigabytes of memory, leading to an immediate Out-Of-Memory (OOM) application crash.
**Learning:** Even if length fields describe data boundaries within an expected binary format, they must be validated against hard security limits or bounded against available resources before satisfying memory allocation requests, because format parsers are prime vectors for DoS.
**Prevention:** Implement strict length checks, such as bounds capping (`Math.min(length, Math.max(0, this.size - offset))`) in file readers and exporting/enforcing a hard security constant (`MAX_SAFE_ALLOCATION`) in the stream readers before processing allocation requests.
