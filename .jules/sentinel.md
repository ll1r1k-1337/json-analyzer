## 2024-03-07 - Denial of Service via Unbounded String Table
**Vulnerability:** The `BinaryTokenWriter` did not enforce any limits on the number of unique strings or the total memory consumed by the string table during binary generation. An attacker providing a maliciously crafted JSON file with an excessive number of unique strings (or exceptionally long unique strings) could exhaust server memory, leading to an Out-Of-Memory (OOM) Denial of Service (DoS). Additionally, calculating string byte length using character count (`.length`) instead of UTF-8 byte length inaccurately tracked memory consumption for multi-byte characters.
**Learning:** In-memory tracking structures (like the string table buffered until finalization) must always have configurable, enforceable upper bounds, especially when processing external or untrusted inputs. Relying on stream processing alone does not mitigate memory risks if sub-components accumulate unbounded state. Additionally, `Buffer.byteLength(value, "utf8")` should be used over `.length` for accurate memory accounting.
**Prevention:** Implement `BinaryWriterOptions` with `maxUniqueStrings` and `maxStringTableBytes` to explicitly bound in-memory resource consumption. Throw a fast-failing security error when these limits are exceeded to immediately halt processing. Use accurate UTF-8 byte length calculation for string memory tracking.
