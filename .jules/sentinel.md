## 2024-05-18 - Out-Of-Memory (OOM) Denial of Service via buffer allocation limit
**Vulnerability:** In `src/binary/reader.ts`, `BinaryTokenReader` parses token lengths from a binary stream (e.g. `byteLength = lengthBytes.readUInt32LE(0)`) and directly allocates memory for those sizes via `Buffer.allocUnsafe` and `Buffer.alloc` without constraints. An attacker supplying a maliciously crafted `.bin` file could specify massive token lengths (up to ~4GB) for strings or typed arrays, causing instant Out-Of-Memory crashes.
**Learning:** External inputs dictating buffer or array lengths must always be constrained within safe application-specific limits, even if the underlying runtime allows larger sizes, as attackers will use maximums to exhaust system memory.
**Prevention:** Implemented a strict `MAX_SAFE_ALLOCATION` limit (512MB) across buffer read operations in `FileReader.read`, `BinaryTokenReader.readBytes`, and `parseStringTable`. Length requests exceeding this limit throw a synchronous error before allocation occurs.
